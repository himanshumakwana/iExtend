# iExtend Logo Design

**Status:** Approved 2026-05-10.

**Goal:** Establish a brand mark for the iExtend iPad app ‚Äî visible on every onboarding screen, in the running-app chrome, in Settings, and as the springboard app icon. Today the iPad app has no logo: the Welcome screen carries only a small text capsule reading "iExtend 1.0".

## Visual concept

**Origami Fold, floating** (concept "L3" in the brainstorm). Three folded paper panels in a left-to-right cascade, gradient-shaded so each panel reads as a distinct surface catching different light. The fold suggests a workspace mid-unfolding ‚Äî a screen extending into another screen, expressed without literally drawing two displays.

The mark sits with a soft floor shadow and an indigo ambient glow behind it, so it appears to float a few millimeters off the surface. The glow opacity adapts to dark/light context; the panel gradients themselves are fixed.

### Geometry

Three quadrilateral panels, anchored at four shared crease points. Coordinates are expressed in a 160√ó160 SVG/SwiftUI viewBox so the in-app SwiftUI mark and the icon SVG share the same source numbers:

| Panel | Vertices (x,y) | Gradient (top ‚Üí bottom-right) |
|---|---|---|
| 1 (left) | `34,42 ‚Üí 88,30 ‚Üí 96,118 ‚Üí 38,108` | `#0a84ff ‚Üí #0050b8` (blue) |
| 2 (mid) | `88,30 ‚Üí 122,52 ‚Üí 122,128 ‚Üí 96,118` | `#5e5ce6 ‚Üí #3f3d80` (indigo) |
| 3 (right) | `122,52 ‚Üí 138,76 ‚Üí 134,134 ‚Üí 122,128` | `#bf5af2 ‚Üí #7a3acd` (purple) |

Crease lines drawn between panels at 0.6pt, white at 50‚Äì70% opacity, sell the fold without dominating it. Panel 1 also carries a top-down white sheen (18% ‚Üí 0%) for surface character.

### Floating effect

- **Floor shadow:** elliptical, blurred (Gaussian œÉ‚âà4), 55% black, centered ~50pt below the geometric center.
- **Ambient glow:** radial gradient (`#5e5ce6` 60% ‚Üí 0%) sized roughly 80√ó50pt behind the panels.
- **Dark mode:** glow at full strength as specified.
- **Light mode:** glow opacity √∑ 2; floor shadow opacity √ó 0.6 (light backgrounds need less contrast).

### Wordmark

"iExtend" in SF Pro Display, weight `.semibold`, tracking ‚âà ‚àí1%. White or `Theme.ink` depending on context. Mark and wordmark are spaced by 0.5√ó the mark's height when laid out horizontally.

## Architecture

### Three deliverables

1. **`LogoMark`** ‚Äî pure SwiftUI view drawing the fold.
2. **`LogoLockup`** ‚Äî `LogoMark` + wordmark, two preset sizes.
3. **App icon** ‚Äî single 1024√ó1024 PNG asset rendered from a canonical SVG.

The in-app marks are drawn live in SwiftUI. The springboard icon must be raster (iOS requirement) and is a build-time render from SVG.

### Component contracts

**`LogoMark`**

```swift
public struct LogoMark: View {
    public init(size: CGFloat = 64, floats: Bool = true)
    public var body: some View { ... }
}
```

- `size` controls the mark's width (height tracks proportionally ‚Äî viewBox is square).
- `floats` toggles the floor shadow + ambient glow. Set `false` for the app-icon SVG render and for tightly-clipped contexts (e.g., the `FloatingToolbar` corner).
- Renders via three `Path` shapes filled with `LinearGradient`, plus a `RadialGradient` halo and a `.shadow` modifier.
- No external dependencies; reads only `@Environment(\.colorScheme)` to scale the glow.

**`LogoLockup`**

```swift
public struct LogoLockup: View {
    public enum Size { case compact, hero }
    public init(_ size: Size = .compact)
    public var body: some View { ... }
}
```

- `.compact` ‚Äî 24pt mark, 14pt wordmark. For headers and toolbars.
- `.hero` ‚Äî 46pt mark, 22pt wordmark. For the Welcome screen.
- Wordmark uses `Theme.ink` for foreground (theme-aware).

**`BrandGradient`**

Extends the existing `LinearGradient.brandGradient` (already used in `WelcomeView.swift:76`). Adds three named static values: `.brandFoldBlue`, `.brandFoldIndigo`, `.brandFoldPurple` ‚Äî one per fold panel. Single source of truth so any future palette shift updates both the headline and the logo.

### App-icon render pipeline

`scripts/app-icon-source.svg` is the canonical source for the springboard icon. The icon background is a 1024√ó1024 rounded square (radius 224, matching iOS 17 icon mask) with a dark indigo‚Üípurple linear gradient (`#1a1530 ‚Üí #2a1a4a`). The fold mark is centered at ~62% width, drawn with the same panel coordinates as `LogoMark`, scaled and translated to fit.

`scripts/generate-app-icon.sh` resolves its own location (via `dirname "$0"`) so it works from any CWD, then calls `rsvg-convert -w 1024 -h 1024 <repo>/scripts/app-icon-source.svg -o <repo>/ipad/iExtend/Assets.xcassets/AppIcon.appiconset/icon-1024.png`. The script verifies `rsvg-convert` is on PATH and prints a one-line install hint (`apt install librsvg2-bin`) if not. Output PNG is committed to the repo ‚Äî the script is run-once when the icon changes, not on every build.

`Contents.json` uses iOS 17+'s single-size template:

```json
{
  "images": [
    {"filename": "icon-1024.png", "idiom": "universal", "platform": "ios", "size": "1024x1024"}
  ],
  "info": {"author": "xcode", "version": 1}
}
```

## Tradeoffs and choices

### Drift between SwiftUI mark and SVG icon

The SwiftUI `LogoMark` and `app-icon-source.svg` both encode the same geometry. They can drift if one is updated without the other.

**Chosen:** Accept the drift. Both files reference the same shared coordinate set in their headers as documentation. Geometry changes are rare.

**Alternative considered:** Render the SwiftUI `LogoMark` to PNG via `ImageRenderer`. Single source of truth, but `ImageRenderer` requires macOS at render time. Current dev box is Linux. Deferred to follow-up once Xcode is on the dev box; would replace `generate-app-icon.sh` with a `swift run` target.

### `rsvg-convert` vs `resvg` vs `magick`

`rsvg-convert` is in Debian/Ubuntu's default repos (`librsvg2-bin`), handles gradients and filters correctly, and produces clean output at 1024√ó1024.

`resvg` is faster and more standards-compliant but isn't packaged in the same way. `magick` (ImageMagick) handles SVG poorly without an additional render backend.

**Chosen:** `rsvg-convert`. Easiest to install, sufficient quality for our geometry (no exotic SVG features used).

### Single-size icon vs legacy matrix

The app already targets iOS 17+, where Xcode accepts a single 1024√ó1024 source asset and resamples for every required size at build time. No reason to ship the 20-icon legacy matrix.

**Chosen:** Single-size template.

### Theme awareness

The mark's panel gradients are *fixed* ‚Äî they don't recolor in light vs dark mode, because the brand identity is the gradient. Only the floating effects (glow, shadow) scale by colorScheme.

**Chosen:** Fixed gradients, theme-aware glow only. Simpler, and matches how other Apple-ecosystem app marks behave (Apple's own apps don't recolor their icons by mode).

## Testing

- **Visual snapshot test (unit):** Render `LogoMark(size: 256, floats: true)` in light and dark color schemes, snapshot to PNG, compare against committed reference. Catches accidental geometry / color regressions.
- **Manual visual check:** Open each modified screen (`WelcomeView`, `DiscoverView`, `PairView`, `PinEntryView`, `LiveView`, `SettingsView`) in SwiftUI Preview. The brainstorm visuals at `.superpowers/brainstorm/.../origami-variants.html` (variant L3) are the reference for "what it should look like".
- **Icon render verification:** After running `scripts/generate-app-icon.sh`, open `icon-1024.png` and compare against `app-icon-source.svg` rendered in a browser at 1024px. They should be visually identical.
- **No automated test for the SVG ‚Üí PNG pipeline.** Output is committed; if `rsvg-convert` produces a bad PNG, it's caught at the visual check step.

## Out of scope

- Animated launch logo (the M√∂bius / Pixel Cascade options were considered for this; not pursued in v1).
- iPadOS 18 light/dark/tinted icon variants (the new triplet system). v1 ships a single icon; tinted variants can be added later by writing two more 1024√ó1024 PNGs.
- Updating any host (Rust daemon, tray, installer) brand chrome ‚Äî this spec is iPad-app-only. The tray's existing branding is untouched.
- macOS / web companions ‚Äî none exist yet.
